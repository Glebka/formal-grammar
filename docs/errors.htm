Сообщения об ошибках

 Недопустимое имя нетерминала.

В имени нетерминала использоавны недопустимые символы. Обратитесь к главе Правила кодирования грамматики.

 Нетерминал 'ИМЯ_НЕТЕРМИНАЛА' был определен ранее.

Данный нетерминал уже встречался ранее в левой части одного из правил грамматики. Воспользуйтесь поиском по тексту грамматики (комбинация клавиш Ctrl + F), найдите первое определение нетрминала и добавьте в конец правой части правила синтаксические термы из правила, которое вызывает эту ошибку (см. Теория языков и формальных грамматик).

 Строка не является правилом грамматики.

Обратитесь к главе Правила кодирования грамматики.

 Недостижимый символ 'ИМЯ_НЕТЕРМИНАЛА'.

Эта ошибка возникает, когда в грамматике определен нетерминал, который не встречается в правых частях правил. Чтобы устранить эту ошибку, удалите правило, определяющее этот нетерминал, или включите его в правую часть соответствующего правила.

 Непроизводящий символ 'ИМЯ_НЕТЕРМИНАЛА'.

Эта ошибка возникает, когда в грамматике обнаружен нетерминал, который встречается только в правых частях правил, но не встречается в левой части ни одного правила, т.е является неопределенным. Чтобы устранить эту ошибку, определите этот нетерминал, т.е создайте новое правило, в левой части которого будет указан этот нетерминал.

 Возможна бесконечная рекурсия при выводе цепочки из символа 'ИМЯ_НЕТЕРМИНАЛА'.

Как правило, таких сообщений может быть несколько. Эта ошибка говорит о том, что следуя правилам грамматики невозможно вывести конечную цепочку символов. Рассмотрим пример грамматики, которая вызывает такую ошибку:

program = header body "end"

char = "a" ! "b" ! "c" ! "d" ! "e"

operator = "оператор" operator

program_name = char ! char program_name

header = "Program " program_name

body = operator ! operator body

Обратите внимание на правило:

operator = "оператор" operator

Очевидно, что это правило - бесконечно рекурсивное. Из-за ошибки в этом правиле бесконечно рекурсивными стали нетерминалы "body" и "program".

Для того, чтобы устранить эту ошибку, можно, к примеру, отредактировать правило так:

operator = "оператор" operator ! "оператор"

 Конфликт типа 'ТИП_КОНФЛИКТА'.

 

Возникновение конфликтов в матрице предшествования свидетельствует о наличии неоднозначностей в грамматике, которые делают невозможным синтаксический анализ методом простого предшествования. Обратитесь к главе Отладка грамматик за информацией о том, как устранить конфликты.

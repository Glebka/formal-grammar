Грамматики простого предшествования

Грамматика называется грамматикой простого предшествования, если она контекстно свободная, обратимая, не содержит пустых правых частей, а матрица предшествования не содержит конфликтов.
Грамматика называется обратимой, если она не содержит одинаковых правых частей.
Пример грамматики простого предшествования:

S -> bBb

B -> (C | a

C -> Ba)

В синтаксически правильно заданной цепочке два соседних символа могут быть связаны между собой с помощью одного из трех специальных бинарных отношений простого предшествования. Это отношения "=" (равно), "< "(меньше) и > (больше).
Нахождение отношений прстого предшествования между символами грамматики необходимо для выполнения синтаксического анализа методом простого предшествования. Обычно отношения простого предшествования представляют в виде квадратной матрицы, которую называют матрицей предшествования.

В матрице простого предшествования строки и столбцы отмечаются символами из полного словаря грамматики. Полный словарь грамматики - это множество терминальных и нетерминальных символов этой грамматики. Для приведенной выше грамматики полный словарь V={S,B,C,b,(,a}. На пересечении строки с номером i и столбца с номером j записывается знак отношения предшествования между символами Хi и Yj или ячейка матрицы остается пустой, если между парой символов нет отношения.

Существует несколько способов определения отношений. Первый способ основывается на использовании синтаксических деревьев, при этом мы последовательно генерируем деревья и путём их анализа определяем отношения. Процесс продолжается до тех пор, пока не будут сгенерированы все типы деревьев. Этот способ применим только к простым грамматикам. Отношения предшествования могут быть вычислены на основе специальных бинарных отношений FIRST, LAST, EQUAL.

Матрица не содержит конфликтов, если для любых двух символов полного словаря существует не более одного отношения предшествования. Неформально, это значит, что в каждой клетке матрицы предшествования должно находится не более одного отношения.

Для синтаксического анализа вводится специальный символ "#", которым обрамляется любая цепочка на синтаксическом дереве. Левая решетка всегда "меньше" последующего символа. Символ предшествующий правой решетке, всегда "больше" неё.
